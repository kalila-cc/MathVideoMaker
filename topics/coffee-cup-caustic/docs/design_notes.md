# 咖啡杯焦散线创作复盘

本话题最终成片路径：

`topics/coffee-cup-caustic/exports/final/CoffeeCupCaustic_final_1080p60_chatgpt_intro.mp4`

这次迭代的核心经验不是某个单独画面，而是“可见现象 -> 反射光线族 -> 包络线 -> 命名曲线”的说明链条。后续做光学、包络线、参数曲线或公式推导类视频时，应优先复用下面的规则。

## 问题与解决方式

### 1. 概念和目标出现得太突然

问题：早期版本直接出现 `x\sin(2\theta)-y\cos(2\theta)=R\sin\theta`、`F(x,y,\theta)=0` 或“焦散线”等术语，观众不知道它们从哪里来，也不知道当前到底在求什么。

解决：在推导前增加“直线族的来源”，明确目标是求反射光线的直线方程，再把每条反射光线统一写成一族方程，为后续包络线条件做准备。术语首次出现时给出可操作定义：“很多反射光线最密的边界”，再补充英文 caustic / caustic curve 和中文“焦散线/焦散曲线”的对应。

复用规则：任何抽象公式第一次登场前，都要先说明三件事：正在求什么、为什么现在要求它、求出来下一步能做什么。

### 2. 反射方向推导跨度过大

问题：从反射规律直接跳到 `\vec d=\vec i-2(\vec i\cdot\vec n)\vec n` 和 `(-\cos2\theta,-\sin2\theta)`，观众看不到法线分量、切线分量和二倍角化简的来源。

解决：拆成“入射方向”“法线方向”“投影系数”“法线分量”“切线分量”“反射时切线不变、法线反向”“代入坐标”“二倍角化简”几个小步，并让画面同步显示分量箭头。

复用规则：向量反射、投影、叉积、行列式、偏导等抽象工具不能只给公式。每个工具都要配一个图形角色，并且一次只推进一个非平凡变换。

### 3. 线方程推导缺少几何构造

问题：隐式直线方程看起来像凭空出现，`P_0`、`\vec m`、`\vec d` 的关系不够清晰。

解决：先在图上给任意点稳定命名为 `P_0 (x,y)`，再画 `P_0-P(\theta)` 位移向量；引入与反射方向垂直的 `\vec m`，展示 `\vec d\cdot\vec m=0`，最后写出点法式并展开。公式推导中保留上一条式子，再添加下一条，让观众能比较变化。

复用规则：当公式使用点、方向向量、法向量或辅助向量时，图中必须同时出现这些对象。若某个箭头只服务前一个小节，进入下一小节时要淡出，避免和新向量抢语义。

### 4. 包络线条件容易被误解

问题：`F=0` 后再写 `\partial F/\partial\theta=0`，容易被理解成“常数求导为零”，而不是包络线的接触条件。

解决：保留简洁的两条条件，但加小字说明：候选亮点 `Q=(x,y)` 固定，`x,y` 不动，只让族参数 `\theta` 变化；偏导为零表达的是邻近反射光线也贴着同一点擦过。

复用规则：包络线条件必须讲清“固定什么、变化什么、为什么要看导数”。不要把偏导说成对恒等式求导。

### 5. 信息密度和旁白时长不匹配

问题：部分右侧面板一次性塞入过多公式，旁白变长后画面静止时间过久，或者画面已经切到下一段而声音还在上一段。

解决：把公式面板拆成多个状态：已知量、换方向、点法式、代入展开、整理成直线族；每段旁白对应一个画面动作或公式新增。最终以实际 SRT/音频长度校准场景时长。

复用规则：长旁白不应对应静态满屏公式。若一个公式面板需要停留超过数秒，必须增加语义运动，或拆成连续的短视觉 beat。

### 6. 符号、读音和坐标约定需要统一

问题：几何角度一开始用 `t` 不够直观；旁白里“行”“量”等多音字被 TTS 读错；坐标公式出现时，部分画面没有同步给坐标轴或极坐标参考。

解决：把几何参数统一改成 `\theta`，后续公式、旁白、字幕一并替换。旁白中避免孤立使用“行”“量”，改成“直线”“变量”“数值”“长度”等更明确说法。出现直角坐标公式时补 `x/y` 轴；出现极坐标方程时补极点、极轴、`\rho` 径向线段和 `\theta` 扫描动画。

复用规则：参数如果实际是角度，优先用 `\theta`。任何坐标形式都必须配同屏坐标参照。TTS 风险词在脚本阶段就要改，不要等成片后再补救。

### 7. 光线方向和反射关系需要显式化

问题：只有一束反射线或一组淡色光线时，观众不一定能看出“入射 -> 反射”的对应关系；有些方向箭头和向量标签又会在后续画面里变成冗余。

解决：在反射光线族动画中保留一条动态高亮的入射光线和对应反射光线，并给当前代表光线加方向箭头。进入点法式推导后，淡出已经完成使命的反射方向箭头，只保留当前需要的 `\vec m` 和位移关系。

复用规则：光学场景默认给当前入射/反射光线加方向提示；但方向箭头不是永久装饰，进入新的数学任务后要清理旧提示。

### 8. 结果展示不能只停在参数式

问题：解出参数式以后，如果只说“这是一条肾形线”，观众仍然缺少形状、标准方程和生成方式的整体认识。

解决：先显示参数式和完整曲线，再补充 nephroid / 肾形线名称、极坐标方程、`\rho` 绕一圈的极坐标动画，以及“小圆绕大圆滚动”生成同一条曲线的动画。解释不同方程只是同一曲线的不同写法。

复用规则：命名曲线的 payoff 至少包含完整形状和一个可视化生成方式；如果补充坐标方程，必须先交代坐标约定和它与前一结果的关系。

### 9. 移动端可读性优先

问题：右侧信息在后半段突然变大或越框，后来又一度偏小，移动端看不清。

解决：把本片确认过的面板字号沉淀为未来 16:9 Manim 数学视频的最小字号基线。密集信息先拆分，只有在无法避免时才缩小。

复用规则：移动端可读性优先于桌面端的“精致感”。密集推导不要靠压字号解决，先靠拆分和分步出现解决。

### 10. 封面和最终清理也属于交付质量

问题：封面如果加厚黄色曲线，会遮挡自然形成的包络线；最终清理只删音视频会留下旧预览封面。

解决：封面保留多条光线自然形成边界，不额外覆盖粗高亮曲线。最终清理只保留最终视频和 metadata 引用的最终封面，旧预览视频、静音母版、Manim 片段、临时音频、旧封面和孤儿 poster 都要清理。

复用规则：封面要展示主题机制，不只是漂亮结果。最终交付检查必须包含视频、封面、metadata 和图库 API 四项。

## 后续同类视频检查清单

- 开头是否从具体可见现象出发，而不是从公式或术语出发？
- 第一个抽象公式出现前，是否说明了当前任务和下一步用途？
- 每个公式符号在图上是否有对应对象、颜色或标签？
- 推导是否一次只推进一个非平凡变换？
- 长旁白是否被拆成多个画面状态，而不是一个静态满屏公式？
- 坐标公式是否有坐标轴，极坐标公式是否有极点、极轴、`\rho` 和 `\theta`？
- 入射/反射光线是否有方向提示，旧箭头是否在任务切换后淡出？
- 命名曲线是否展示完整形状、方程关系和至少一种生成方式？
- TTS 风险词是否在脚本阶段规避？
- 最终输出是否只保留最终视频和最终封面，并确认图库只展示目标成片？
