# 贝塞尔曲线选题大纲 v1

## 选题

**钢笔工具里的数学：贝塞尔曲线到底怎么拐弯？**

## 观众承诺

看完以后，观众能同时得到两层理解：

- 感性层：贝塞尔曲线是由一组“点在点之间移动”的过程画出来的。
- 理性层：控制点怎样精确影响曲线，可以由权重、导数和拼接条件计算出来。

## 章节结构

1. **几个点，凭什么管住一条曲线？**
   - 画面：矢量编辑器里的锚点、手柄和曲线。
   - 核心疑问：控制点不全在曲线上，却能控制曲线形状。

2. **先理解一段线上的移动点**
   - 画面：点 A、点 B 和参数 `t`。
   - 公式：`L(t)=(1-t)A+tB`。
   - 解释：`t` 是进度，公式是沿线段做加权平均。

3. **三点曲线：先分点，再分点**
   - 画面：`P_0, P_1, P_2` 的 de Casteljau 构造。
   - 中间桥：先写两条第一层移动点，再代入第二层移动点，展开并合并同类项。
   - 公式：`B(t)=(1-t)^2P_0+2(1-t)tP_1+t^2P_2`。
   - 解释：`2(1-t)t` 来自 `P_1` 在左右两条线段里各出现一次；三次的 `1,3,3,1` 是同一规则多展开一层后的结果。

4. **四点曲线：钢笔工具常用的版本**
   - 画面：三次 de Casteljau 构造，显示 `P_0` 到 `P_3`。
   - 公式：`B(t)=(1-t)^3P_0+3(1-t)^2tP_1+3(1-t)t^2P_2+t^3P_3`。
   - 解释：这是三次贝塞尔曲线，设计软件里的路径通常用它表示。

5. **拖一个控制点，曲线到底怎么算着变？**
   - 画面：动态权重条，拖动 `P_1` 后曲线局部变形。
   - 公式：`P_i` 移动 `\Delta`，曲线点移动 `w_i(t)\Delta`。
   - 解释：控制不是模糊拉扯，而是每个位置按权重响应。

6. **手柄为什么决定出发和到达方向？**
   - 画面：起点、终点切线与手柄方向重合；两段曲线平滑拼接。
   - 公式：`B'(0)=3(P_1-P_0)`，`B'(1)=3(P_3-P_2)`。
   - 拼接条件：`P_3=Q_0` 且 `P_3-P_2=Q_1-Q_0`。

7. **编程里的贝塞尔曲线**
   - 画面：CSS `cubic-bezier`、SVG `C` 指令、Canvas API、动画缓动曲线。
   - 核心回响：程序存的不是很多像素，而是控制点和计算规则。

## 理解风险

- 如果公式太早出现，观众会以为这是纯代数推导。先用构造动画建立直觉，再压缩成公式。
- 系数必须展示来源，不能只标注“权重”：二次先展开出 `P_1` 的两项再合并，三次只点明 `1,3,3,1` 来自多一层展开。
- “控制点拉动曲线”不能停留在比喻，要补上权重函数和控制点位移的精确响应。
- 端点手柄方向需要用导数解释，否则观众只能记住经验规则。
- 编程段不要扩展成 API 教程，只建立“图形、动画、字体、路径数据都在用同一种数学对象”的联系。
