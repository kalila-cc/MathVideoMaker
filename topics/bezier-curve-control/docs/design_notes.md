# 贝塞尔曲线视频设计记录

## 视觉分区

- 问题观察：动态曲线、控制多边形和信息面板如果共享同一块画面区域，低清预览里会出现曲线穿过面板、点位被遮住或数值贴边裁切的问题。
- 后续规则：带有右侧公式/权重面板的镜头，需要先划定动画安全区和信息安全区；控制点移动后的极限位置也必须留在动画安全区内。
- 检查方式：分段渲染后抽取该段前中后多帧，确认动画轨迹、面板、公式、数值读数没有互相覆盖。

## 预览迭代

- 问题观察：每修一个分段就同步拼接完整 MP4，会让完整预览的修改时间晚于分段预览，审片网页容易把完整预览排到前面，误导当前工作重心。
- 后续规则：贝塞尔曲线低清审片阶段以分段预览为主；单段反馈只重渲染对应 Scene，等整体章节节奏基本稳定后再重新拼完整视频。
- 问题观察：改用 Remotion/JS 重做某个分段后，如果只导出新 MP4 但没有同步更新 `data/videos.json` 的分段路径，预览网页会继续加载旧的 Manim 段落，造成“代码已改但画面没变”的误判。
- 后续规则：替换单个分段的渲染实现时，必须同时更新分段预览 metadata；如果分段预览外挂整条旁白，替换段落本身应使用无声 MP4，避免双音轨。
- 检查方式：本地图库中分段预览标记为当前审片版本，完整预览只作为阶段性连看产物保留。

## 旁白发音

- 问题观察：`P 二对后半段...` 这类标签后接“对”的句式，中文 TTS 容易连读成“P 两对...”。
- 后续规则：控制点标签后面不要直接接容易组成数量词的虚词；紧凑词如 `后半段` 也可能被拆成奇怪断句。优先改成 `P 二在曲线后面...` 或 `靠近终点的一侧...`。
- 检查方式：改旁白后只更新分段预览外挂 MP3/SRT，完整 MP4 等整体预览稳定后再统一重拼。

## 公式严谨性

- 问题观察：控制点位移响应段如果只写 `\Delta B(t)=w_i(t)\Delta P_i`，观众会不确定这里是单个控制点的特例，还是多个控制点同时移动的通式。
- 后续规则：多个控制点可能同时移动时必须写求和式 `\Delta B(t)=\sum_i w_i(t)\Delta P_i`；如果画面只拖一个控制点，要额外标出单点特例。
- 检查方式：公式、动画动作和旁白对象必须一致；比如只移动 `P_1` 时，画面可以同时展示通式和 `\Delta B(t)=w_1(t)\Delta P_1`。

## 点符号标识

- 问题观察：后半段镜头里公式已经出现 `P_3=Q_0`、`P_3-P_2=Q_1-Q_0`、`B'(0)=3(P_1-P_0)` 等符号，但画面如果只显示曲线和手柄，观众无法把公式里的点映射回图形。
- 后续规则：凡是旁白或公式点名 `P_i`、`Q_i`、`B(t)`，同一镜头必须给对应点或移动点加清楚的符号标识。标签要避开曲线、箭头文字和公式面板。
- 检查方式：渲染分段后抽关键帧，逐段确认画面上的点标签覆盖了公式里出现的符号。

## Remotion 包装方向

- 可用判断：Remotion 适合做本片的后期包装层，不适合在当前阶段替代 Manim 的核心数学动画。曲线构造、公式展开和控制点运动仍由 Manim 保持精确。
- 优先优化：统一章节片头、章节进度、公式 callout、关键术语字幕、轻量转场、背景细纹理、片尾署名，以及未来横版/竖版重排。
- 暂缓事项：在分段预览还频繁微调时，不建立完整 Remotion 成片链路，避免每次反馈都要同步维护 Manim 片段、Remotion 时间线和完整 MP4。
- 落地时机：等 10 个分段的内容、旁白、符号标注和画面遮挡基本稳定后，再搭建 `topics/bezier-curve-control/remotion/`，用 Remotion 读取已批准的 Manim 片段和旁白音轨做整体包装预览。

## ChatGPT 片头

- 问题观察：最终发布版的 ChatGPT 标识不应临时改成透明角标；本项目已有幂和主题验证过的片头标识语言，复用它能保持系列一致性。
- 后续规则：贝塞尔最终高清版使用 `power-sum-polynomials` 同款 `ChatGPTIntro`：绿黑面板、线性 ChatGPT 标识、制作说明和四个制作范围标签。片头作为独立静音片段放在正片前，音频按片头实测时长整体后移。
- 检查方式：每次重渲染片头都重新测量实际时长；本次 1080p60 片头为 5.05 秒，封装后用 `silencedetect` 验证开头静音区，再检查 5 秒后是否进入原正片。

## 旁白与动画同步

- 问题观察：平滑拼接段旁白从“手柄排成一条直线”推进到“更严格一点，导数向量相等”时，画面如果仍停在同一条直线状态，观众会看不出“更严格”的新增条件。
- 后续规则：旁白引入更强条件时，动画必须出现对应状态变化。本段先展示 `P_3-P_2 \parallel Q_1-Q_0`，再在“导数向量相等”处移动 `Q_1` 到 `Q_1-Q_0=P_3-P_2`，同时画出两条等长导数向量。
- 检查方式：抽取该句字幕附近的关键帧，确认公式、点位和向量箭头同时从“方向相同”切换到“方向和长度都相同”。

## 编程章节的渲染方式

- 问题观察：编程领域章节需要让控制点、曲线和 `cubic-bezier(...)` 参数数字同步变化。用 Manim 逐帧重建动态文字、公式和曲线时，1080p60 渲染成本过高；同时把代码块放在曲线框内会遮挡控制点和路径。
- 后续规则：这类“参数驱动的代码示例”优先用 Remotion 原生 SVG/React 实现，曲线用 SVG `path d="M... C..."` 直接绑定 JS 数值，代码数字和控制点共享同一组状态。曲线框内只放曲线、点、手柄和必要标签；代码示例放在框外的独立信息区，避免占用动画安全区。
- 问题观察：如果右侧又展示另一条 SVG/Canvas 曲线，会稀释“CSS cubic-bezier 参数映射”的讲解目标，也会让观众误以为进入了另一个示例。
- 后续规则：编程章节的右侧区域在 CSS 语境内持续展示 `cubic-bezier(x1,y1,x2,y2)`、四个动态数值和 `P1/P2` 坐标映射；不要同时再放第二条曲线。底部公式不再使用圆角信息框，直接作为无框文字层显示，避免压缩公式空间。
- 检查方式：先抽静帧确认信息区完全离开曲线框，再渲染 5 秒样本测速。当前 Remotion 原生 1080p60 样本 300 帧约 8.43 秒，明显快于 Manim 动态文字版。
- 问题观察：如果把 Remotion 段做成完整独立画面，进入编程章节时背景色、光效和卡片体系会突然变化，像切到了另一支视频；英文状态词如 `fixed` 也会削弱中文科普的统一感。
- 后续规则：非 Manim 生成的局部动态内容应继承 Manim 段的底色、面板高度和中文 UI 文案，只把高频变化的曲线、参数条和代码数字交给 Remotion 绑定状态；左右卡片保持同顶同高，端点状态写成“固定”等中文表达。
- 检查方式：替换单段后至少抽取动态初段、中段和参数变化较大处三帧，确认背景不跳色、两侧卡片等高、中文文案完整、曲线形变幅度足够明显。
- 问题观察：右侧代码区如果保留 `transition-timing-function:` 这类旁支属性名，会把注意力从 `cubic-bezier(...)` 的四个参数映射上拉走。
- 后续规则：编程章节只展示当前讲解必需的代码片段和参数关系；底部数学公式继续使用 Times New Roman 风格，和前面 Manim 数学排版保持一致。
- 问题观察：只按 Manim 源码色值复制到 Remotion 不一定能得到同样观感；Manim 片段经过视频编码后，上一章节背景实际接近 `#05101D`，而 Remotion 直接使用 `#07111F` 会显得偏亮偏蓝。
- 后续规则：跨渲染器接续时要抽取相邻章节的实际视频帧作为色彩参考。Remotion 编程段背景按相邻 Manim 帧的解码观感校准，去掉独立网格/光效；曲线、控制点和公式标签使用 Manim 常量色，点标签与底部公式使用 Times 系斜体，代码仍使用 Consolas。

## 背景网格

- 问题观察：全片背景加网格时，如果先直接更新视频，格子大小、正方形一致性和线条可见度的判断成本很高；复用同名静帧还可能让预览界面显示旧图，误判为“没有变化”。
- 后续规则：背景网格先用单帧场景确认，再同步到正式场景；预览图使用唯一文件名。当前确认参数为 `#8FA8D8`、`stroke_width=1.6`、`opacity=0.155`、`spacing=0.48`。
- 检查方式：更新成片前抽取主章节和混合编程章节各一帧，确认两处网格颜色、间距和可见度一致。

## 成片复盘

- 有效解决的问题：编程章节从独立 Remotion 整屏改成 Manim 背景壳 + Remotion 透明局部覆盖，解决了章节背景跳色；右侧代码区去掉旁支属性名，端点状态改中文，底部公式保持 Times 系斜体。
- 有效解决的问题：开篇长旁白不再只停在静态曲线图，`P_1`/`P_2` 增加了语义相关的移动；旧点残影通过移除原始 mobject family 后再添加动态层解决。
- 有效解决的问题：封面规则从桌面/移动两套改为单张移动端优先高清封面，标题按信息流缩略图可读性设计，不再生成持久 200px 检查图。
- 有效解决的问题：背景网格通过单帧候选逐步确认可见度和间距，最终再同步到片头、主章节和混合编程段，避免整片重复渲染后才发现观感不对。
- 有效解决的问题：高清并行渲染时 `QuadraticConstruction` 触发 Manim 文本 SVG 缓存竞态；单独串行补渲染解决。后续应避免在 `always_redraw` 中反复创建不变文本，或为并行 Manim 场景隔离 `media_dir`。
- 最终归档规则：成片确认后，`exports` 只保留最终 MP4 和当前封面；旧预览、静音中间版、Manim 片段、Remotion 透明层、调试帧、旧封面和海报都清理，并同步删除图库中的旧 metadata 条目。
