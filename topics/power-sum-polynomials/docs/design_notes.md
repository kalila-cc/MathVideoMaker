# 幂和公式主题设计记录

## v2 采纳的反馈

- 开头配音不再贴着第一帧进入：v2 音轨和 SRT 整体后移 1 秒，让封面先被看见。
- 去掉“求和机器”和“门/齿轮”比喻：改成“共同线索”和“系数地图”，避免观众需要先理解一个额外的造词。
- `k^0`、`k^1`、`k^2` 等数学标签统一用 `MathTex`，不让公式混进普通中文 `Text`。
- 差分画面从不明含义的圆角矩形，改成带括号的“前 n-1 项累计 + 新增 n^p = 前 n 项累计”。
- 三次方柱状图先只排列柱体，再给标签，保证灰色后续项和前三项底部对齐。
- 差分展开页的说明文字左对齐到公式栈，避免低清预览中越过面板边界。
- 三角数平方页增加“同一个三角数，再乘自己”的箭头和边长标签，不再让 `T_n^2` 方框孤立出现。
- 伯努利数收尾改成系数地图，不再用门和齿轮；ChatGPT 片尾 footer 恢复为复用文案 `Math Video Maker`。
- v2 后续修正：中文说明句不再用 `t2f` 把行内字母数字切到 Times New Roman，避免 `n`、`0` 这类字符在中文基线之上漂浮；公式仍用 `MathTex`。
- v2 后续修正：三角数平方页删掉 `T_n \times T_n` 点阵方框，改为用箭头标出右侧代数公式里的 `n(n+1)/2` 正好等于左侧三角数 `T_n`，避免误导成几何乘法证明。
- v2 后续修正：同一页进一步去掉跨栏箭头和说明文字，只保留左侧 `n(n+1)/2=T_n` 与右侧被框出的括号，降低重复解释和视觉干扰。

## 后续规则

- 幂和主题的视觉语言优先使用表格、系数账、括号、累积条和地图，不使用机械装置类隐喻。
- 封面采用“幂和公式 / 自己算”双行大标题作为主信息，再用 `\sum k^3 = T_n^2` 和底部公式做数学识别点；避免右侧窄面板、小字副标题和大块空白。
- 当前封面标题改用按宽度铺开的方式，而不是继续加固定倍率；手机双列缩略图里应优先保证“幂和公式 / 自己算”可读。
- ChatGPT 标识从片尾移到片头，并改回此前片尾同款的动画出现方式；片头只说明 ChatGPT 完成了推导脚本、动画设计、配音字幕和后期合成，不再先解释选题归属。
- 片头制作范围标签需要给足阅读时间：逐个出现不要太快，全部出现后至少停留约 1 秒再退场。
- 封面作为独立素材生成，不再依赖正片第一帧；本题同时维护 desktop 与 mobile/feed 两版封面并由图库按视口选择。
- 局部改片头或单个 scene 时，优先使用图库 `segments` 分段预览条目检查整体节奏，再决定是否生成完整低清/高清成片。
- 如果要把内容自然引向伯努利数公式，组合数递推应放在待定系数法已经算完之后：先让观众看到系数法可行，再回头用二项式差分说明 `S_p(n)` 可以由 `S_0(n)` 到 `S_{p-1}(n)` 递推得到。
- 组合数递推不能只把 `4,6,4,1` 标成组合数；需要补上“从四个括号里选几个 1”的计数来源，避免观众误以为只是巧合。
- 同一段里也不能一开始就把 `4,6,4,1` 当标题或入口；应先展示 `(k+1)^4` 的展开式，再减去 `k^4`，最后才把留下的 `4,6,4,1` 命名为组合数。
- “望远镜求和”这类术语或比喻在本题里不够直观；优先直接展示逐项抵消，例如 `(2^4-1^4)+(3^4-2^4)+...` 最后只剩首尾项。
- 从单点等式跳到幂和记号时，必须显式展示“对等式两边同时从 `k=1` 加到 `n`”这一桥接，再用 `S_p(n)=\sum k^p` 的定义把求和符号换成 `S_p(n)`。
- 如果在组合数递推后首次说出“伯努利数”，旁白必须立刻给出一句基础定义，例如“它是一串有理数，用来整理高次幂和里反复出现的固定系数”，不能只丢出名字或公式。
- 如果后续再单独做伯努利数章节，重点应变成知识扩展：幂和来源、泰勒展开、欧拉麦克劳林求和公式、黎曼泽塔函数等。只需简单承认它和前面递推有关，避免把整页强行讲成前文推导的角色分工。
- `S_3(n)=T_n^2` 可以作为公式彩蛋，但独立成段会分散“系数法到组合数递推再到伯努利数”的主线；当前版本删去平方 payoff 段。
- 后续如果重做封面，主标题需要按手机双列视频流缩略图验收：缩到约 160 到 200 像素宽时，“幂和公式”“自己算”这类核心字仍要能看清。
- v9 字体试用：得意黑 / Smiley Sans 适合封面和大标题，标题识别度明显更强；它只有单一窄斜字重，Manim/Pango 在 Windows 上不能强行请求 `BOLD`，实验版使用官方 TTF 并去掉 `weight=BOLD`。正文小字是否全部使用得意黑还需要看预览后再定。
- 低清抽帧必须覆盖：差分示意、待定系数页、差分展开页、组合数递推桥接、伯努利数扩展页、ChatGPT 片头。
- 开头动机应先从“只背现成求和公式，会在下一条公式前停住”切入，再转向“把已有答案放在一起，看是否存在同一套结构”；避免“公式长出来”这类拟人化说法，旁白要直接说明结构观察和待定系数问题。
- 本题 TTS 预览使用 `--rate +14%` 作为默认语速；改动开头旁白后，要用新 SRT 的章节边界同步调整前后相邻 scene 的 `wait_to` 总时长，不能只替换音频，否则分段预览会从中段开始逐渐错位。
- Scene 页眉应使用中性的内容概括，说明“这一段正在讲什么”；避免把口号、比喻或旁白里的口语句直接放进标题和副标题，例如“拿来背”“多一块”“账本”“锁住”这类词更适合旁白，不适合段落页眉。
- 如果调整旁白进入时间，必须同步处理音频和 SRT，避免画面章节与字幕时间脱节；带 ChatGPT 片头时，最终高清封装用 1080p 片头实际时长，分段预览用 480p 片头实际时长，不能沿用旧的 `audioDelay`。

## 最终复盘

- 最终主线确定为“已知低次求和公式的多项式结构 → 待定系数法算三次幂 → 二项式差分推出组合数递推 → 伯努利数知识扩展”。`S_3(n)=T_n^2` 只适合做旁支彩蛋，独立成段会打断这条主线。
- 组合数递推页必须保留三步因果链：先展开 `(k+1)^4`，再减去 `k^4`，再对 `k=1` 到 `n` 求和并按定义替换为 `S_p(n)`。任何一步省略都会让观众觉得结论突然出现。
- 伯努利数第一次出现时只给基础定义；最后章节承担扩展知识面，提到它和前面递推有关即可，不把它包装成前文每个步骤的唯一答案。
- 本题最终使用得意黑 / Smiley Sans 强化封面和大标题，但该字体只有单一官方字重，Manim 里不要再请求 `BOLD`。正文和行内公式仍要优先保证基线和清晰度。
- 本题最终片头是 ChatGPT 制作说明，文案聚焦“本视频制作完全由 ChatGPT 完成”和具体工作项，例如推导脚本、动画设计、配音字幕、后期合成；不要额外解释选题归属。
- 本题同步基准：TTS 使用 `--rate +14%`；高清片头实际约 5.05 秒，低清分段预览片头实际约 5.2 秒。以后重渲染片头后必须重新测量，不能直接沿用旧数值。
- 最终交付后只保留最终高清 MP4、desktop/mobile 封面、最终旁白稿和最终 SRT；清掉旧预览、Manim 中间产物、debug frames、posters 和过期 metadata。
