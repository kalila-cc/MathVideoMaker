把几条最常见的求和公式并排写在纸上，会出现一个小小的怪现象。一串 1 加起来等于 n；一加二一直加到 n，等于 n 乘 n 加一，再除以二；平方和也有一个三次多项式。它们看起来不只是答案，反而像被同一条线索串起来的不同版本。

把这些公式展开看，味道会更明显。零次方的和是一次多项式；一次方的和是二次多项式；平方和是三次多项式。于是我们自然会问：如果换成三次方、四次方，前 n 项和会不会也只是一个更高一次的多项式？如果是，系数能不能自己算出来？

关键是换一个角度看求和。前 n 项和，比前 n 减一项和，只多了最后这一项。也就是说，相邻两个求和结果的差，正好等于 n 的 p 次方。这个动作叫差分。它像给一排积木多推上一块；总量变了多少，就看新来的那一块有多大。

现在拿三次方做实验。我们想求一的三次方，加二的三次方，一直加到 n 的三次方。因为差分会让多项式的次数降一阶，所以能差出三次方的东西，最自然先猜成四次多项式。写成 a 乘 n 的四次方，加 b 乘 n 的三次方，加 c 乘 n 的平方，加 d 乘 n，再加 e。又因为前零项和是零，所以 e 直接是零。

接下来只做一件事：让这个多项式在 n 这里的值，减去它在 n 减一这里的值，等于 n 的三次方。先看四次方这一块，它的差分会产生四倍 n 的三次方、负六倍 n 的平方、四倍 n，以及负一。三次方、平方和一次方也各自差出一串更低次数的项。把同类项排齐，就能按 n 的三次方、n 的平方、n、常数项，一列一列比较系数。

比较以后，四个系数会被依次锁住。四倍 a 等于一，所以 a 是四分之一；平方项要消失，所以 b 是二分之一；一次项也要消失，所以 c 是四分之一；最后常数项要消失，所以 d 是零。于是三次方的前 n 项和，就是四分之一 n 的四次方，加二分之一 n 的三次方，再加四分之一 n 的平方。

这个结果还可以收起来。前三项正好合成 n 的平方，乘 n 加一的平方，再除以四。也就是，一的三次方加到 n 的三次方，等于括号里 n 乘 n 加一除以二的平方。三次方的和，居然是普通等差求和公式的平方。

更有意思的是，这套办法不是三次方专用。只要愿意把系数账本继续往下做，四次方、五次方、更高次的幂和，都能这样算。数学里有一串专门管理这些系数的数字，叫伯努利数。它们像一张系数地图，把所有幂和公式统一写进同一种语言。今天我们只是看见了这张地图的一角。
