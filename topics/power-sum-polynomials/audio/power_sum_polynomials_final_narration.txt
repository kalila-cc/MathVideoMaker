很多求和公式，我们第一次见到的时候，通常都是直接被告诉答案。

比如，一串一加起来是 n；一加二一直加到 n，是 n 乘 n 加一，再除以二；平方和，又突然变成了一个三次多项式。

如果只是把它们当成公式背下来，那下一条三次方求和、四次方求和，就只能继续等别人告诉我们。

但把这几条答案放在一起看，会出现一个更有用的问题：这些公式背后，会不会有同一套结构？

零次方的和，是一次多项式；一次方的和，是二次多项式；平方和，是三次多项式。

也就是说，要求 p 次方的和，也许可以先从一个 p 加一次的多项式开始。

所以轮到三次方的时候，我们不妨先猜：它的前 n 项和，应该是一个四次多项式。

真正的问题就变成了：这个多项式里的系数，能不能不靠背，直接算出来？

关键是换一个角度看求和。前 n 项和，比前 n 减一项和，只多了最后这一项。也就是说，相邻两个求和结果的差，正好等于 n 的 p 次方。这个动作叫差分。它像给一排积木多推上一块；总量变了多少，就看新来的那一块有多大。

现在拿三次方做实验。我们想求一的三次方，加二的三次方，一直加到 n 的三次方。因为差分会让多项式的次数降一阶，所以能差出三次方的东西，最自然先猜成四次多项式。写成 a 乘 n 的四次方，加 b 乘 n 的三次方，加 c 乘 n 的平方，加 d 乘 n，再加 e。又因为前零项和是零，所以 e 直接是零。

接下来只做一件事：让这个多项式在 n 这里的值，减去它在 n 减一这里的值，等于 n 的三次方。先看四次方这一块，它的差分会产生四倍 n 的三次方、负六倍 n 的平方、四倍 n，以及负一。三次方、平方和一次方也各自差出一串更低次数的项。把同类项排齐，就能按 n 的三次方、n 的平方、n、常数项，一列一列比较系数。

比较以后，四个系数会被依次锁住。四倍 a 等于一，所以 a 是四分之一；平方项要消失，所以 b 是二分之一；一次项也要消失，所以 c 是四分之一；最后常数项要消失，所以 d 是零。于是三次方的前 n 项和，就是四分之一 n 的四次方，加二分之一 n 的三次方，再加四分之一 n 的平方。

系数法已经算出答案了。现在回头看差分，还能看到一条更系统的路。不要先猜未知系数，而是直接看相邻四次方的差：k 加一的四次方，减去 k 的四次方。

先把 k 加一的四次方单独展开。它等于 k 的四次方，加四倍 k 的三次方，加六倍 k 的平方，加四倍 k，再加一。然后再减去 k 的四次方，最前面的 k 的四次方被抵掉，留下的系数才是四、六、四、一。

这些数也不是巧合。k 加一的四次方，意思是四个 k 加一相乘。要得到 k 的三次方，就要从四个括号里选一个一，其余三个选 k，所以有四种选法；要得到 k 的平方，就要选两个一，所以有六种选法。继续数下去，就得到四、六、四、一，也就是组合数。

现在关键的一步来了。因为这个等式对每一个 k 都成立，所以可以把等式两边同时从 k 等于一加到 n。左边加起来会逐项抵消，只剩 n 加一的四次方，和最开始减掉的一。

右边也要一项一项加起来。四倍 k 的三次方从一加到 n，就是四倍三次幂和；六倍 k 的平方从一加到 n，就是六倍平方和；后面的一次幂和、零次幂和也一样。按这个定义，右边才会变成四倍 S 三，加六倍 S 二，加四倍 S 一，再加 S 零。

于是三次幂和，不一定要从头猜。把这个式子里的最高次幂和单独移出来，它就可以借用已经知道的低次幂和算出来。

这件事如果换成 p 次方，也一样成立。n 加一的 p 加一次方再减一，等于一串组合数，加权乘上从零次到 p 次的幂和。把最高那一项移到左边，就得到一个递推关系：高次幂和，可以由所有低次幂和一步一步推出。

这里会自然碰到一个名字：伯努利数。它们是一串有理数，专门用来整理高次幂和公式里反复出现的固定系数。刚才这条递推，只是它们登场的一条入口。

如果把视野放宽一点，伯努利数不只服务于幂和公式。它最早和雅各布伯努利系统研究幂和有关，后来还出现在泰勒展开、欧拉麦克劳林求和公式，以及黎曼泽塔函数这些地方。

可以把它粗略理解成一组校正系数：当数学想把离散的求和、连续的积分、函数的展开联系起来时，它们常常会冒出来。今天我们没有完整推导伯努利数，只是从三次方求和这里，看到它为什么值得被单独命名。
