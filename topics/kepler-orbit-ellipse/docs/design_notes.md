# 行星轨道视频设计记录

- 开头不能直接提“椭圆”。本片的观众视角是跟着牛顿式的推理走：先看到太阳和行星的相对位置、初速度和指向太阳的加速度，再问“这样的运动最后会是什么形状”。椭圆应在推导后期作为结果自然出现。
- 面积定律只依赖中心加速度方向，不能提前混入反平方强度。推出面积变化率恒定后，要明确补上“这就是开普勒第二定律”的介绍，并让左侧动画真实表现近太阳处速度更快、远太阳处速度更慢。
- 反平方关系进入形状推导时，要写清它是新加入的牛顿引力规律：由太阳引力造成的加速度大小与距离平方成反比。不要用“太阳按反平方吸引”这类省略了物理量的模糊说法。
- 径向加速度和二阶常微分方程求解不作为本片主推导展开。画面可以直接列出“本片直接使用的结论”，旁白说明完整推导较长、不在这里展开，然后立刻说明它把“指向太阳的加速度”翻译成 `r` 随 `theta` 变化的问题。
- 形状方程的最终表达采用标准轨道元素，而不是把 `r_{\min}`、`r_{\max}` 当成最终常用写法。本片最终保留 `r=a_{\rm orb}(1-e^2)/(1+e\cos\theta)`，并说明 `a_{\rm orb}` 是半长轴、`e` 是偏心率；`k` 在太阳引力中替换为 `GM_\odot`。
- 推导得到 `r=(h^2/k)/(1+e\cos\theta)` 后，要先明确它已经是椭圆的极坐标形式，并用自然中文说明坐标约定：把太阳所在的焦点当作极点，`\theta` 从近日点方向算起。再继续把 `k` 换成 `GM_\odot`，并改写为常用轨道元素。
- 介绍 `e` 时不要说“等认出曲线以后才成为偏心率”这类因果倒置的话。先说它在当前公式里的作用：控制余弦项相对常数项的大小、也控制轨道偏离圆的程度；等标准轨道元素出现时再把它命名为偏心率。
- 结论面板不要把同一条不等式在标题和公式里重复展示。若空间足够，用小分类表补充 `e=0`、`0<e<1`、`e` 接近 1 的形状含义，再保留焦点和速度结论。
- 收尾的真实行星结论卡不要使用 `\Rightarrow \mathrm{ellipse}` 这种英文箭头公式，会像未解释的推导残片。改用中文短标题搭配 `0<e<1`，再补“e 越大轨道越扁、太阳在焦点、面积定律仍成立”；非必要时也少用箭头符号，避免观众误以为这里还有一步推导。
- 周期可以作为现实观测里确定半长轴的补充说明：在太阳质量已知时，周期通过开普勒第三定律对应半长轴。但周期本身不能决定偏心率，也不能单独确定轨道形状。
- 最终公式替换段只展示必要结果，不堆边界方程和中间解法。若需要提示“常数项如何回到真实物理量”，用分阶段画面：抽象式、`k=GM_\odot`、标准轨道元素式；最终画面只保留结论和少量定义。
- 面积定律推导中，从 `\vec r\times\vec v` 守恒转到 `r^2 d\theta/dt` 守恒时必须有桥接段：先取叉乘大小，再说明只使用速度中垂直于半径的部分 `v_t`，并展示 `v_t=r(d\theta/dt)`。不能把极坐标形式当作已经显然成立的下一行公式。
- 初速度变化的彩蛋要逐项和旁白绑定：低速坠落、椭圆、圆、抛物线、双曲线各自在对应句子附近出现并稳定，不允许整组动画提前播放完再靠等待补齐总时长。
